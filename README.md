# Интеллектуальный юридический ассистент СДА (RAG)

Диалоговый ассистент для навигации по нормативно-правовой базе Сретенской духовной
академии (80+ локальных актов + федеральное законодательство). Построен по схеме
**Retrieval-Augmented Generation**: семантический поиск по корпусу документов +
генерация ответа моделью **YandexGPT** со ссылкой на конкретный нормативный акт.

🔗 **Демо:** https://supernewrag-cyzi3lf3qj3s7q8is7mwnh.streamlit.app/

> Курсовой проект, ОП «Искусственный интеллект в маркетинге и управлении продуктом»,
> ФКН НИУ ВШЭ.

## Возможности

- **Трёхступенчатый итеративный поиск** (`iterative_search`): семантический поиск
  (k=12) → поиск по ключевым словам (k=5 × 3) → кросс-документный анализ ссылок (k=4);
  из ~47 кандидатов в модель уходят топ-15 фрагментов.
- **Ответ из 4 блоков**: заключение → правовое обоснование → порядок действий →
  документы; плюс уточняющие вопросы-кнопки.
- **Smart-роутер шаблонов**: типовые запросы (академотпуск, отчисление, справка и т. п.)
  отдают готовый `.docx` из `docs/templates/`.
- **Защита от галлюцинаций**: гиперссылки берутся только из реестра `docs/site_index.json`;
  age-guardrail добавляет выверенное примечание методиста по вопросам о возрасте/приёме.
- **Суверенный контур (152-ФЗ)**: корпус, FAISS-индекс и SQLite — локально; генерация —
  через российский YandexGPT, без передачи данных за рубеж.

## Технологический стек

| Слой | Технология |
|------|-----------|
| Интерфейс | Streamlit 1.x |
| Векторный поиск | FAISS (LangChain Community), 16 446 фрагментов |
| Эмбеддинги | `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2` (384) |
| Генерация | YandexGPT (`gpt://{FOLDER_ID}/yandexgpt/latest`, t=0,2) |
| Хранение истории | SQLite3 |

## Структура репозитория

```
app.py                  — основное Streamlit-приложение (UI + RAG + роутер шаблонов)
requirements.txt        — зависимости
sretensk_db/            — готовый FAISS-индекс корпуса (index.faiss, index.pkl)
docs/
  site_index.json       — реестр разрешённых гиперссылок
  templates/            — 25 .docx-шаблонов заявлений
  *.png, *.html         — ассеты интерфейса
eval/                   — воспроизводимая оценка качества (см. ниже)
```

## Запуск локально

```bash
pip install -r requirements.txt
# секреты YandexGPT — через переменные окружения или .streamlit/secrets.toml
export YANDEX_API_KEY=<ваш_ключ>
export FOLDER_ID=<ваш_folder_id>
streamlit run app.py
```

> `.env`, `*.db` и `venv/` намеренно исключены из репозитория (`.gitignore`).
> SQLite-база с историей диалогов не коммитится: она привязана к анонимному
> идентификатору сессии и не содержит персональных данных в смысле 152-ФЗ.

## Оценка качества (воспроизводимо)

Папка `eval/` содержит размеченный тест-сет и скрипты, измеряющие систему на **тех же**
параметрах поиска, что и `app.py`.

```bash
cd eval
python retrieval_eval.py     # метрики поиска → retrieval_results.csv
YANDEX_API_KEY=<ключ> python e2e_eval.py   # сквозная оценка ответов → answers_for_grading.csv
```

| Метрика | Значение |
|---------|----------|
| Recall@15 (поиск, 32 вопроса) | **78,1%** (25/32) |
| MRR | 0,546 |
| Точность цитирования | 87,5% (28/32) |
| Соблюдение структуры ответа | 90,0% (36/40) |
| Точность среди данных ответов | 96,4% (27/28) |
| Корректный отказ на граничных | 7/8 |

Recall@15 вплотную подходит к потолку ~77%, зафиксированному для русскоязычного
юридического RAG (RuTaR, Alibekov et al., 2025).

**Файлы оценки:**
- `eval/test_questions.csv` — 40 вопросов (32 по корпусу + 8 граничных) с эталонными источниками;
- `eval/retrieval_results.csv`, `eval/answers_for_grading.csv` — готовые результаты прогонов;
- `eval/survey_instrument.md`, `eval/survey_results.csv` — анкета и данные пилотного
  опроса пользователей (N=5): средняя полезность 9,0/10, точность 8,4/10, намерение
  пользоваться 8,6/10.
